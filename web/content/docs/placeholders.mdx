<Callout type="info" title="Cores">
  O SpawnerX aceita códigos de cor com `&` (ex.: `&a`, `&c`, `&6`).
</Callout>

## Config.yml

### `spawner.display-name` e `spawner.lore`

| Placeholder | Descrição |
| --- | --- |
| `{type}` | Nome amigável do mob (ex.: Zumbi) |
| `{rarity}` | Texto da raridade (sem cor) |
| `{rarity_color}` | Código de cor da raridade |
| `{stack_size}` | Stack interno do spawner |

> O plugin sempre prefixa `{stack_size}x` no display name.

## Locale (locale/*.yml)

### Comandos

| Chave | Placeholders |
| --- | --- |
| `commands.give.success` | `{amount}`, `{type}`, `{player}` |
| `commands.give.received` | `{amount}`, `{type}` |

## shop.yml (lore customizada)

No `shop.yml`, você pode definir `lore` customizada por item usando:

- `{price}`
- `{currency}`
- `{type}`
- `{display_name}`

Use `\n` para quebrar linhas dentro de `lore` em string única.

<Callout type="warn" title="Prefix">
  O placeholder `{prefix}` é substituído automaticamente em todas as mensagens de locale.
</Callout>
